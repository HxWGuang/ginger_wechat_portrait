# 微信聊天记录分析工具 · 安装指南

> ⚠️ **使用前须知**：本工具通过读取你自己设备上的微信本地数据库来分析你自己的聊天记录，
> 全程在本地运行，AI 人格分析由 Claude Code 本身完成，不需要任何 API Key，不向外部发送消息内容。
> **请勿在他人设备上操作，请勿用于分析他人数据。**

---

## 哪些步骤是自动完成的

运行 `/analyze-wechat` 命令后，以下步骤**全部自动处理**，无需手动操作：

- 安装 Python 依赖库
- 下载并配置微信数据库解密工具（`wechat-db-decrypt-macos`）
- 解密微信数据库（已有密钥文件时）
- 自动查找解密后的数据库路径并写入配置
- 导出指定联系人的消息及昵称 / 头像信息
- 生成可视化图表
- 由 Claude Code 完成 AI 人格分析（双人对比）
- 生成完整 HTML 报告并在浏览器打开

## 需要你手动完成的部分（共 2 步，均为一次性）

---

### 第 1 步：关闭 SIP

SIP（System Integrity Protection，系统完整性保护）是 macOS 的安全机制，会阻止调试器附加到微信进程提取密钥。**只需关闭一次**，密钥提取完成后即可重新开启。

**操作步骤：**

1. 完全关机

2. 进入恢复模式：
   - **Apple Silicon（M 系列芯片）**：按住电源键不放，直到出现"正在加载启动选项"画面 → 点击**选项**
   - **Intel 芯片**：开机时按住 `Command + R`

3. 进入恢复模式后：顶部菜单栏 → **实用工具** → **终端**

4. 执行以下命令并重启：
   ```
   csrutil disable
   reboot
   ```

5. 重启后在普通终端中验证：
   ```bash
   csrutil status
   # 应显示：System Integrity Protection status: disabled
   ```

---

### 第 2 步：手动运行密钥提取脚本

微信 4.1.x 改变了加密机制，原有的 `find_key.py`（基于 lldb 断点）**已不适用**。现改用 **C 内存扫描器**直接扫描微信进程内存，搜索密钥特征字符串。运行时需要 sudo 权限。

**运行 `/analyze-wechat` 后，Skill 会自动完成构建**，你只需在 Terminal.app 中执行扫描命令。

1. 打开系统自带的 **Terminal.app**（在启动台 → 其他 → 终端）

2. 复制 Skill 提供的命令，粘贴到 Terminal.app 执行（形如）：
   ```bash
   sudo /tmp/wechat-decrypt/find_all_keys_macos
   ```

3. 脚本会输出扫描进度和匹配结果，自动保存为 `all_keys.json`

4. 回到 Claude Code，告诉 Skill 已完成，后续转换和解密全自动

> **常见问题：**
> - 报 `task_for_pid failed`：需要 sudo 运行，或先对微信做 ad-hoc 重签名：
>   ```bash
>   sudo codesign --force --deep --sign - /Applications/WeChat.app
>   ```
>   重签名后重启微信，可不需 sudo 运行扫描器
> - 报 `setCipherKey function not found`：说明 Skill 尝试了旧版 lldb 方案，请告知 Skill 使用 C 扫描器替代
> - **微信 4.0 以下版本**仍可使用旧版 `find_key.py`（lldb 断点方式），Skill 会先尝试旧方案，失败后自动切换到 C 扫描器

---

## 密钥提取完成后：立即重新开启 SIP（推荐）

密钥文件（`wechat_keys.json`）保存后，解密操作就不再需要 SIP，**建议立刻恢复系统安全性**：

1. 重启进入恢复模式（同第 1 步 1～3）
2. 终端执行：
   ```
   csrutil enable
   reboot
   ```

**重新开启 SIP 不影响已有的密钥文件和解密后的数据库，之后所有分析完全不需要 SIP。**

---

## 第二次及之后使用

只要 `wechat-db-decrypt-macos/decrypted/` 目录下存在 `.db` 文件，Skill 会**直接跳过所有解密步骤**，整个流程完全自动，你只需输入联系人名字。

如果解密数据库被清理，但密钥文件（`wechat_keys.json`）还在，Skill 会自动重新解密，无需 SIP 也无需手动操作。

## 常见问题

**Q：脚本运行时微信卡住很久，正常吗？**
A：C 内存扫描器**不会卡住微信**——它只是读取进程内存。如果用的是旧版 lldb 方式才会卡住，建议切换至 C 扫描器。

**Q：解密后找不到数据库文件**
A：在微信里随便打开几个对话，等几秒让消息写入磁盘，再重新运行 Skill。

**Q：出现 `setCipherKey function not found`？**
A：你用的是微信 4.1+ 版本，原有 lldb 方案已失效。Skill 会自动切换到 C 内存扫描器方案，只需运行一次 `sudo` 命令即可。

**Q：自己消息数为 0 / 全部消息都显示为对方发的？**
A：微信数据目录名中的 hash 后缀（如 `_078f`）与数据库中的 wxid 不匹配。Skill 已修复此问题，更新到最新代码即可。

**Q：凌晨消息的日期显示为前一天？**
A：时区转换问题已修复。如果仍出现，确认 `data_loader.py` 中使用了 `tz_convert('Asia/Shanghai')`。

**Q：Apple Silicon 报架构错误**
A：C 内存扫描器不涉及 Python 架构问题，直接编译为原生 ARM64 二进制。

**Q：不想分析对方，只想看自己的数据**
A：正常运行即可，如果没有对方的分析结果，报告会以单人模式展示。
