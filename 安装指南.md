# 微信聊天记录分析工具 · 安装指南

> ⚠️ **使用前须知**：本工具通过读取你自己设备上的微信本地数据库来分析你自己的聊天记录，
> 全程在本地运行，AI 人格分析由 Claude Code 本身完成，不需要任何 API Key，不向外部发送消息内容。
> **请勿在他人设备上操作，请勿用于分析他人数据。**

---

## 哪些步骤是自动完成的

运行 `/analyze-wechat` 命令后，以下步骤**全部自动处理**，无需手动操作：

- 安装 Python 依赖库
- 安装 sqlcipher 和两个外部工具：
  - [`Thearas/wechat-db-decrypt-macos`](https://github.com/Thearas/wechat-db-decrypt-macos) — 数据库解密（`decrypt_db.py`）
  - [`ylytdeng/wechat-decrypt`](https://github.com/ylytdeng/wechat-decrypt) — 密钥提取（C 内存扫描器 `find_all_keys_macos`）
- 解密微信数据库（已有密钥文件时）
- 自动查找解密后的数据库路径并写入配置
- 导出指定联系人的消息及昵称 / 头像信息
- 生成可视化图表
- 由 Claude Code 完成 AI 人格分析（双人对比）
- 生成完整 HTML 报告并在浏览器打开

## 需要你手动完成的部分（仅 1 步，一次性）

---

### 第 1 步：提取微信数据库密钥

微信聊天数据库使用 SQLCipher 加密，首次使用需要从微信进程内存中提取密钥。工具使用来自 [ylytdeng/wechat-decrypt](https://github.com/ylytdeng/wechat-decrypt) 的 **C 内存扫描器**（`find_all_keys_macos`），通过 Mach VM API 直接读取进程内存，**不需要关闭 SIP**（与旧版 lldb 方案不同）。

**运行 `/analyze-wechat` 后，Skill 会自动 clone 该仓库到 `/tmp/wechat-decrypt` 并编译**，你只需在系统 Terminal.app 中执行一条命令：

1. 打开系统自带的 **Terminal.app**（启动台 → 其他 → 终端）

2. 复制 Skill 提供的命令执行（形如）：
   ```bash
   sudo /tmp/wechat-decrypt/find_all_keys_macos
   ```
   首次需要输入 sudo 密码，扫描约 10 秒完成。

3. 扫描结果自动保存为 `all_keys.json`，回到 Claude Code 告知完成，后续解密全自动。

#### 可选：避免每次 sudo（给微信做 ad-hoc 重签名）

如果不想每次使用 `sudo`，可对微信做一次 ad-hoc 重签名：

```bash
sudo codesign --force --deep --sign - /Applications/WeChat.app
```

重签名后**重启微信**，之后运行扫描器无需 sudo。

> **说明**：微信默认启用了 [Hardened Runtime](https://developer.apple.com/documentation/security/hardened_runtime)，阻止其他进程读取其内存。重签名移除该限制后，扫描器无需特权即可访问。微信更新后需重新执行重签名。

#### 常见问题

| 错误信息 | 处理方式 |
|---------|---------|
| `task_for_pid failed: 5` | 用 `sudo` 运行，或先对微信做 ad-hoc 重签名 |
| `setCipherKey function not found` | 旧版 lldb 方案不适用，Skill 会自动切换至 C 扫描器 |
| 编译失败 | 安装 Xcode Command Line Tools：`xcode-select --install` |
| 微信更新后扫描器不工作 | 重新执行 ad-hoc 重签名命令 |

## 第二次及之后使用

只要 `wechat-db-decrypt-macos/decrypted/` 目录下存在 `.db` 文件，Skill 会**直接跳过所有解密步骤**，整个流程完全自动，你只需输入联系人名字。

如果解密数据库被清理，但密钥文件（`wechat_keys.json`）还在，Skill 会自动重新解密，无需手动操作。

## 常见问题

**Q：扫描器运行时微信会卡住吗？**
A：不会。C 内存扫描器只读取进程内存，不附加调试器，微信全程正常使用。

**Q：需要关闭 SIP 吗？**
A：不需要。新版使用 C 内存扫描器 + sudo（或 ad-hoc 重签名），与 SIP 无关。只有旧版 lldb 方案才需要关闭 SIP。

**Q：出现 `setCipherKey function not found`？**
A：微信 4.1+ 移除了该函数。Skill 会自动切换至 C 内存扫描器。

**Q：自己消息数为 0 / 全部消息显示为对方发的？**
A：微信数据目录名的 hash 后缀（如 `_078f`）与数据库中的 wxid 不匹配。更新到最新代码即可解决。

**Q：凌晨消息的日期显示为前一天？**
A：时区转换问题已修复。更新到最新代码即可。

**Q：解密后找不到数据库文件**
A：在微信里打开几个对话，等消息写入磁盘后重新运行。

**Q：分析多个人会互相覆盖吗？**
A：不会。输出按联系人自动创建独立子目录（`wechat_analysis_output/<联系人名>/`），互不干扰。

**Q：Apple Silicon 兼容性**
A：C 扫描器编译为原生 ARM64 二进制，Python 脚本通过 venv 隔离运行，均完全兼容。
